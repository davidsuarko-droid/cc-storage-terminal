## Корзина-накопление (заказ как инвентарь Factorio) (2026-06-18)
Тап по плитке копит количество (бейдж `+N` латунью), бар корзины снизу со сводкой
`Cart N items / M units` + `[Confirm] [Clear] [Step:N]`. Confirm шлёт всю корзину одним
проходом. Шаг накопления циклится тапом по `Step`: 1 → 8 → 64 (тач-монитор без модификаторов).
- `src/ui_logic.lua` — `basketNew/Add/Qty/List/Totals` (чистые, кламп [0,count], порядок), `nextStep`.
- `src/render.lua` — бейдж `+N` на плитке, латунная рамка корзинных плиток, бар Confirm/Clear/Step + хит-зоны.

## Сеть: сервер + покет-клиент (2026-06-18)
Стенной монитор И портативный advanced-pocket. `startup.lua` = диспетчер: глобал `pocket` есть → клиент.
- `src/net.lua` — НОВЫЙ: протокол `ccstore` поверх rednet. Чистые строители `reqStock/respStock/
  reqOrder/respOrder/kind` (тесты) + I/O `net.open` (находит модем по типу).
- `src/server.lua` — НОВЫЙ: стационарный. Тикер+монитор+модем. Тач-заказ (как раньше) + `serveLoop`:
  отвечает на `{t=stock}` списком товаров+адресов, на `{t=order}` зовёт `order.place` → `{got}`.
- `src/pocket.lua` — НОВЫЙ: клиент. Тянет сток у сервера (broadcast→первый ответ), рисует на `term`,
  заказ **мышью**: ЛКМ=стак(64), ПКМ=1шт, колесо над плиткой=±шаг, колесо вне плитки=листать.
  Поиск с клавиатуры. Палитра/рендер общие с сервером (`render.draw` surface-agnostic, `setTextScale` guard).
- Покет-модем встроенный; крафт с ender pearl = ender-дальность (бесконечно).

## Грид-редизайн Create/стимпанк (2026-06-18)
Список-строки → **вид магазина**: сетка плиток-слотов (спрайт + счётчик + id), степпер количества.
Скин — Create/стимпанк (андезит-корпус + латунь-акцент), перекрывает белый DESIGN.md
(тот для сайтов; здесь монитор в MC + явная просьба на стимпанк).
- `src/sprites.lua` — НОВЫЙ: сикстант-символы CC (128-159), 1 клетка = 2×3 субпикселя.
  `M.cell` чистый энкодер (BR — опорный субпиксель для инверсии fg/bg), юнит-тест.
  7 спрайтов категорий 4×6 (Create=шестерёнка, Redstone=факел, …). `M.draw` = I/O.
- `src/render.lua` — переписан: палитра Create через `setPaletteColour` (андезит-серые +
  латунь/медь/парчмент + 7 цветов бейджей), грид плиток (beveled-корпус, спрайт цветом
  категории, `xN` счётчик, id снизу trunc), горизонт-чипы, степпер-кейпад оверлей
  (`- + +8 +64 Max Clear OK X`), индикатор `page X/Y`. **English ASCII** (CC-шрифт без кириллицы).
  `render.applyPalette(mon)` зовётся в startup.
- `src/ui_logic.lua` — `layout` (chips+grid на всю ширину), `gridDims`, `tiles`, `chips`, `stepper`.
- `src/startup.lua` — perPage-скролл, тач плиток/чипов, степпер-логика, `model.pressed`, палитра.
- `scripts/gen-installer.lua` — НОВЫЙ: переиспользуемый генератор бандла (читает `src/*.lua` +
  `*.cfg`, подбирает уровень длинных скобок). Запуск из корня: `lua5.1 scripts/gen-installer.lua`.
- Анимация тонкая: латунная рамка нажатой плитки, инверсия, скролл на страницу
  (попиксельный плавный скролл не делаем — лаг синка монитора в MP).
- Лимит движка: нет текстур MC на мониторе → спрайты на **категорию** (7), не на предмет.
- Глаз-чек раскладки: `lua5.1 test/render-preview.lua [W H]` (палитра/спрайты в превью не видны —
  мок не красит, sprites.draw подменён ASCII-плейсхолдером; цвета проверять in-game).

## Дальше — in-game проверка (за пользователем)
Расстановка сервера: рядом с компом — Stock Ticker, склад, Advanced Monitor, Wireless/Ender Modem.
Деплой: `wget run <raw installer>` (один бандл на сервер и на покет — `startup.lua` сам определяет
устройство), либо `src/*.lua` + `*.cfg` **плоско** в корень компа, либо в `world/computercraft/computer/<id>/`. Ребут.

Чек-лист:
1. Скин: тёмный андезит-фон, латунные акценты, плитки-корпус (не мусор-глифы).
2. Грид ~6-8 колонок: спрайт категории + счётчик `xN` + id снизу, читаемо.
3. Чипы категорий в ряд (Create первым после All); тап фильтрует, активный = латунь.
4. Остатки обновляются ~2с.
5. Тап по полю поиска → фокус (латунь); печать фильтрует живьём; backspace стирает; enter снимает фокус.
6. Тап по полосе адреса переключает Main↔Core.
7. Скролл `[^]`/`[v]` листает страницами (`page X/Y`).
8. Тап по плитке копит в корзину (бейдж `+N`); тап `Step` циклит шаг 1→8→64;
   `Confirm` шлёт всю корзину (тост «Sent M units…»); `Clear` чистит.
9. Нажатие плитки → латунная рамка/инверсия (тонкий фидбэк), без лагов.
10. **Покет**: установить тот же installer на Advanced Pocket Computer (встроенный wireless-модем);
    к серверу подключить Wireless/Ender Modem. Покет тянет сток; ЛКМ=стак, ПКМ=1шт,
    колесо над плиткой=±шаг, колесо вне=листать; Confirm шлёт заказ серверу по rednet.

## GPU-рендер: пивот на Tom's Peripherals (2026-06-19)

Реализованы `render_text.lua` (переименован из `render.lua`) и `render_gpu.lua` (новый).
Автоселект бэкенда в `server.lua`: `peripheral.find("tm_gpu")` есть → GPU-рендер (Tom's Monitor,
ARGB-пиксели, полные имена, крупный шрифт, иконки); нет → text-рендер (старое поведение,
CC Advanced Monitor, 16 цветов). Покет (`pocket.lua`) всегда использует `render_text`.

### Что готово
- `render_gpu.lua` — плитки с пиксельным bevel, крупный drawText, корзина-прокрутка, иконки
  категорий (7 категорий, хардкод). Событие тача `tm_monitor_touch` (x, y, sneaking).
- `icons.lua` — LRU-кэш реальных PNG-иконок (лимит 64): `idToFile`, `parseLayer0`, `icons.get`.
  Фолбэк на категорию при 404 или отсутствии файла.
- `src/ui_logic.lua` расширен `layoutPx(wpx, hpx)` — зоны в пикселях для GPU-бэкенда.
- `nextStep4`: цикл 1/16/32/64 (GPU-шаг по умолчанию 32; sneak+тап = +16).
- `test/mock-gpu.lua` — стабы Tom's GPU; `test/preview-gpu.lua` — текстовый дамп вызовов
  для глаз-чека без real hardware.
- Тесты: 154 юнит-теста зелёные, smoke OK, GPU-preview печатает плитки/имена/hit-сводку.
- `installer.lua` перегенерирован: 16 lua + 2 cfg, включает `render_text`, `render_gpu`, `icons`.

### Реальные иконки предметов — отложено (требует ручного шага)
Для настоящих PNG-иконок нужно запустить `scripts/build-icons.py` против папки `mods/`
сервера (SFTP: `[[reference-minecraft-server-sftp]]`) и закоммитить заполненную папку `icons/`.
В рантайме при отсутствии файла `icons/<ns>__<name>.png` — автоматический фолбэк на
пиксель-иконку категории (7 штук). Сборки без `icons/` остаются зелёными.

### Known limitations / TODO (in-game тюнинг, из спека)
- **Маппинг моделей упрощён**: `build-icons.py` v1 парсит только `layer0` item-модели на
  1 уровень. Предметы с многослойными/родительскими/3D-моделями (большинство блоков Create)
  не получат реальной иконки → фолбэк на категорию. Доработать: рекурсивный parent-разбор
  + (опц.) headless-рендер 3D-моделей в иконки (как JEI-дамп).
- **Ленивый wget без оффлайн-режима**: если GitHub raw недоступен / нет сети у сервера —
  иконки не подгрузятся (фолбэк на категорию). Доработать: опция «весь пак в installer»
  или предзагрузка пака одним архивом.
- **decodeImage-формат PNG** подтверждён по исходникам Tom's Peripherals — проверить in-game
  на реальном GPU (размер/масштаб `drawImage`, мультипликатор `setSize`).
- **VRAM/диск-лимиты CC** при сотнях иконок — настроить размер LRU и `maxVRAMSize` под факт.
- **Размеры пикс-плиток и кегли шрифта** подбираются in-game: превью-мок не красит и
  не меряет шрифт; значения 60×72 px/плитку и size=1.0 — отправная точка, тюнить вживую.
- **`setSize(resolution)`** — разрешение экрана GPU зависит от физического размера
  multiblock Tom's Monitor; `getSize()` вернёт реальные пиксели после первого `refreshSize()`.
- **Узкий монитор (1 колонка)** → `perPage=1`; не критично, но можно добавить компактный
  layout-режим.
- **sneak-инкремент (+16)** работает через `tm_monitor_touch` sneaking-флаг; на стандартном
  CC-мониторе sneak недоступен → Step-цикл 1→8→64 остаётся единственным способом.
