# Дизайн: GPU-пивот рендера + реальные иконки предметов (Tom's Peripherals)

Дата: 2026-06-19
Проект: cc-storage-terminal
Статус: одобрен, готов к плану реализации

## Контекст и проблема

Терминал-маркетплейс над Create Stock Ticker рисует сток на CC-мониторе символьной
сеткой (16 цветов, текст + сикстант-спрайты категорий). Две боли пользователя:

1. **Корзина переполняется** — при наборе многих позиций видно только 2 строки + «+K more»,
   остальное не видно.
2. **Текста/иконок недостаточно** — по обрезанным именам и спрайту-категории непонятно
   «что есть что». Дизайн хороший, но различимость предметов низкая.

На сервере (Skybound SMP, NeoForge 21.1.233) установлен **Tom's Peripherals** — добавляет
**GPU-периферию** с настоящим пиксельным рендером: ARGB-цвет, масштабируемый текст,
рисование картинок. Это снимает оба ограничения (на мониторе сервера).

## Ключевые факты о Tom's Peripherals GPU (проверено по исходникам)

- Периферия тип **`tm_gpu`** → `peripheral.find("tm_gpu")`.
- Рисует на **своём** мониторе (блок `toms_peripherals:monitor`, крафтится ИЗ
  `computercraft:monitor_advanced`), собранном в сетку (multiblock-экран).
- GPU находит смежную сетку мониторов автоматически (`refreshSize()`), одной гранью наружу.
- Тач-событие **`tm_monitor_touch`** с параметрами `(x, y, sneaking)` — флаг приседания
  даёт «второй клик» на мониторе (раньше на CC-мониторе невозможно).
- Графика (GPU2D, см. wiki GPUImpl): `filledRectangle`, `rectangle`, `line`, `lineS`,
  `drawText(x,y,text,[fg],[bg],[size],[pad])`, `drawTextSmart`, `drawImage(x,y,ref)`,
  `drawBuffer(x,y,w,scale,argb...)`, `newImage`, `imageFromBuffer`, `decodeImage`, `fill`,
  `getSize` (пиксели), `setSize(resolution)`, `sync()`.
- Цвет: ARGB `0xAARRGGBB` либо R,G,B[,A].
- **`decodeImage` = `NativeImage.read` (STB) → принимает PNG-байты напрямую**
  (`util/NativeImageIO.java`). Значит иконки шипим как обычные сжатые PNG.
- **Нет нативного `drawItem`/рендера MC-моделей** — реальные иконки готовим оффлайн.

## Цели («лучше ванильного склада Create»)

1. Постоянная видимая корзина с накоплением (Create-тикер так не показывает).
2. Реальные иконки + полные имена + поиск + категории — мгновенно понятно что есть.
3. Пакетный заказ: много позиций → один адрес → один Confirm.
4. Крупный чёткий GPU-рендер (ARGB, не 16 цветов).
5. Покет-версия (мобильный заказ с руки) — у Create нет.

## Архитектура: split рендера

Чистая `ui_logic.lua` (layout, tiles, basket, filter, page) — **общая для монитора и покета,
не трогаем по сути** (логика оперирует прямоугольниками, без разницы пиксели/символы).
Расходится только слой рисования:

```
ui_logic.lua  (pure: layout/tiles/basket/filter/page)   ← общий
   │
   ├── render_text.lua   ← переименование текущего render.lua.
   │       Символьный CC term/monitor. Покет + фолбэк сервера (нет tm_gpu).
   │
   └── render_gpu.lua    ← НОВЫЙ. Tom's GPU, пиксели. Только сервер с tm_gpu.
```

- `server.lua`: `peripheral.find("tm_gpu")` есть → backend = `render_gpu` (Tom's monitor);
  нет → `render_text` на CC-мониторе (текущее поведение, старые сборки не ломаются).
- `pocket.lua`: всегда `render_text`.
- Бэкенд = единый интерфейс: `backend.draw(target, model) -> hit`. Тач-роутинг
  (`handleTouch` по `hit`-прямоугольникам) общий.

`ui_logic` получает параллельный `M.layoutPx(wpx, hpx)` — та же структура зон, но пиксельная
сетка с крупными плитками (иконка ~32px + полное имя). Хелперы `tiles/page/basket*`
переиспользуются без изменений.

## GPU-рендер (render_gpu.lua)

**Палитра:** стимпанк-цвета (андезит/латунь/медь) как ARGB-константы. `setPaletteColour`
не нужен — полноцвет, мягкие bevel/градиенты.

**Плитка:**
```
+------------------+   bevel-рамка латунь, если позиция в корзине
|  [ICON]   x128   |   ICON = drawImage реальной текстуры 32×32 (или иконка категории)
|  [32x32]    +8   |   x128 = сток (мелкий ARGB-текст), +8 = бейдж корзины (латунь)
|  Cogwheel        |   ПОЛНОЕ имя крупным читаемым шрифтом, перенос на 2 строки
+------------------+
```

**Тач + инкременты:**
- обычный тап по плитке = **+32** (полстака, ходовой объём Create).
- **sneak+тап = +16** (точная докладка).
- Step-кнопка переключает объём по тапу: цикл **1 / 16 / 32 / 64**.
- Кнопки Confirm/Clear/Step, чипы категорий, поиск, адрес — по hit-зонам, как сейчас.
- Покет (text): колесо над плиткой = ±step; ЛКМ/ПКМ как сейчас.

**Корзина (фикс переполнения):**
- Панель корзины на крупном мониторе показывает много строк; при нехватке — **своя
  прокрутка** (стрелки в шапке корзины, отдельные от грид-скролла).
- Строка: `Qty x Name` + мелкая иконка. Видно весь набор.

## Пайплайн иконок

**Оффлайн — `scripts/build-icons.py`** (разово + при добавлении модов):
- Источник джарок: папка `mods/` сервера, тянем по SFTP в локальный кэш
  (creds: `[[reference-minecraft-server-sftp]]`).
- Из джарки: `assets/<ns>/models/item/<name>.json` → если модель типа `item/generated`
  (есть `textures.layer0`) → достаём PNG `assets/<ns>/textures/<layer0>.png`.
- Если модель `parent` = блок-модель (3D, нет `layer0`) → **пропускаем** (фолбэк на
  иконку категории в рантайме).
- Выход: `icons/<ns>__<name>.png` (16×16, как есть, сжатый PNG) + `icons/manifest.txt`
  (список доступных id).
- Пак коммитим в репо (публичный) → раздача через GitHub raw.

**Рантайм (сервер, GPU-бэкенд):**
- Чистая функция id→filename: `create:cogwheel` → `create__cogwheel.png` (двоеточие → `__`).
- Ленивая загрузка: иконка нужна и нет в кэше → `wget` PNG с GitHub raw в `/icons/<file>`
  → `decodeImage(bytes)` → image-ref в памяти (LRU, лимит ~64, `image.free()` при вытеснении).
- Нет файла / 404 / 3D-блок → пиксель-иконка категории (7 шт, хардкод в `render_gpu`).
- На диске CC только PNG (~200 байт/иконка); VRAM — через LRU+free.

## Детект и деплой

- `startup.lua`: глобал `pocket` → `pocket.lua`; иначе → `server.lua`.
- `server.lua`: `tm_gpu` есть → `render_gpu`; нет → `render_text` (CC monitor).
- Деплой: тот же `wget run <installer url>`. Installer везёт `render_gpu.lua`.
  Иконки НЕ в installer (ленивый wget). Старт мгновенный.

## Что скрафтить/поставить (in-game)

- **GPU Chip (Raw)** — верстак `r a r / g c g / r d r`
  (r=редстоун-пыль, a=аметист, g=золото, c=медь по центру, d=алмаз).
- **GPU Chip** — переплавить Raw в печке (200 тиков).
- **GPU-блок** — верстак `I b I / r c r / I C I`
  (I=железо, b=железные прутья, c=gpu_chip, C=медь, r=редстоун-пыль).
- **Tom's Monitor** ×N — верстак `I m g / L m g / I m g`
  (I=железо, m=`computercraft:monitor_advanced`, g=стеклянная панель, L=редстоун-лампа);
  1 крафт = 2 монитора, собрать в сетку (напр. 3×3).
- Сборка: Advanced Computer ← GPU-блок (смежно или wired modem); GPU-блок ← сетка Tom's-
  мониторов (лицом наружу). Плюс как раньше: Stock Ticker + склад + Wireless/Ender Modem.

## Тестирование

- `ui_logic` — юниты как есть; добавить `layoutPx` (зоны не пересекаются, плитки влезают).
- Маппинг иконок — юнит чистой функции `id→filename` + парсер `layer0` на фикстуре model.json.
- `render_gpu` — мок Tom's GPU (`test/mock-gpu.lua`: стабы `filledRectangle/drawText/
  drawImage/decodeImage/sync/getSize`) + превью-дамп вызовов (текстовый снимок) для глаз-чека.
- Смоук: `startup` грузится при наличии и отсутствии `tm_gpu`.
- `build-icons.py` — прогон на нескольких джарках: плоские item-текстуры извлекаются,
  3D-блоки пропускаются.

## Порядок реализации (фазы)

1. **Рефактор**: `render.lua`→`render_text.lua`, бэкенд-интерфейс, детект `tm_gpu` (фолбэк).
   Старое поведение сохранено.
2. **`render_gpu.lua`**: плитки/крупный текст/тач/корзина-скролл + **иконки категорий**
   (без реальных текстур). Уже большой выигрыш в читаемости.
3. **Пайплайн иконок**: `build-icons.py` + ленивая загрузка + `decodeImage`. Реальные текстуры.
4. **Полировка**: корзина-скролл, sneak-инкременты, тюнинг кеглей/размеров in-game.

## Known limitations / TODO (доделать позже)

- **Маппинг моделей упрощён**: в v1 парсим только `layer0` item-модели на 1 уровень.
  Предметы с многослойными/родительскими/3D-моделями (большинство блоков Create) НЕ получат
  реальной иконки → фолбэк на категорию. Доделать: рекурсивный разбор parent-цепочки +
  (опц.) headless-рендер 3D-моделей в иконки (как JEI-дамп).
- **Ленивый wget без оффлайн-режима**: если GitHub raw недоступен/нет сети у сервера —
  иконки не подгрузятся (фолбэк на категорию). Доделать: опция «весь пак в installer» или
  предзагрузка пака одним архивом.
- **decodeImage-формат** подтверждён как PNG по исходникам — проверить in-game на реальном
  GPU (размер/масштаб `drawImage`, ресурс-мультипликатор `setSize`).
- **VRAM/диск-лимиты** CC при сотнях иконок — настроить размер LRU и `maxVRAMSize` под факт.
- **Размеры пикс-плиток/кегли** подбираются in-game (превью-мок не красит и не меряет шрифт).
